Estadística Espacial + Machine Learning Clásico

Mini proyectos aplicados al contexto colombiano, integrando datos de área, geoestadística, aprendizaje automático clásico y comunicación estadística.

Instrucciones generales

Cada grupo desarrollará un mini proyecto aplicado que combine herramientas de Estadística Espacial y Machine Learning clásico, usando datos reales del contexto colombiano. El trabajo debe evidenciar comprensión metodológica, manejo adecuado de datos espaciales, validación de modelos e interpretación crítica de resultados.

Entregables

  • Documento tipo artículo, máximo 10 páginas.
  • Repositorio reproducible con código y datos procesados.
  • Presentación oral de 10 a 15 minutos.
  • Anexos técnicos si son necesarios.

Componentes mínimos

  • Mapa exploratorio inicial.
  • Análisis de autocorrelación espacial.
  • Modelo clásico de Machine Learning.
  • Modelo o técnica espacial comparativa.
  • Interpretación sustantiva del fenómeno.

Estructura sugerida del documento

  1. Resumen y palabras clave.
  2. Introducción y problema de investigación.
  3. Descripción de datos y fuentes.
  4. Análisis exploratorio espacial.
  5. Metodología estadística y de Machine Learning.
  6. Resultados, mapas, métricas y comparación de modelos.
  7. Discusión crítica.
  8. Conclusiones, limitaciones y trabajo futuro.
  9. Referencias.

Geoestadística + ML regresivo

Producción agrícola, clima y predicción regional

Contexto

La producción agrícola depende de condiciones climáticas, geográficas, económicas y territoriales. Este proyecto busca predecir producción o rendimiento agrícola municipal para un cultivo específico en Colombia.

Objetivo

Modelar la producción o rendimiento agrícola municipal usando covariables climáticas, geográficas y socioeconómicas, evaluando patrones espaciales de predicción y error.

Metodología sugerida

EDA espacial por cultivo.
Random Forest, XGBoost o regresión regularizada.
Kriging de residuos.
Moran sobre residuos.
Importancia de variables y dependencia parcial.

Preguntas orientadoras

  • ¿Qué variables explican mejor el rendimiento agrícola?
  • ¿Hay regiones donde el modelo falla sistemáticamente?
  • ¿El kriging de residuos mejora el desempeño?
  • ¿Qué limitaciones tienen los datos agrícolas reportados?

Datos sugeridos

  • Evaluaciones Agropecuarias Municipales o datos agrícolas abiertos.
  • Variables climáticas del IDEAM u otras fuentes abiertas.
  • Altitud, precipitación, temperatura o distancia a centros de mercado.
  • Shapefile municipal.

Producto esperado

Mapa de rendimiento observado, predicción municipal, residuos espaciales e interpretación agroterritorial.

Recomendación técnica

Incluyan una validación tradicional y una validación espacial. Comparen si el modelo mantiene buen desempeño cuando se evita mezclar territorios vecinos entre entrenamiento y prueba.
